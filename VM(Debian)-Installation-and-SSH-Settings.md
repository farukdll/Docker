  * ###### Enter your username and password and open the debian graphical interface.
  * ###### `sudo apt update`  → The command downloads and updates the latest updates and package lists from software repositories on a debian-based system.
  * ###### We get the error "is not the sudoers file." because we are not authorized to use the sudo command.

  <h1 align="center">
  <p>
      <img height="300" width="450" src="https://github.com/farukdll/Docker/assets/97880185/f6d87202-84dc-42cc-9455-283b54053d62">

  * ###### Let's use the "su" command to become root (super user).
  * ###### To use the "sudo" command, use nano /etc/sudoers to access the "/sudoers/" file in the "/etc/" directory.
  * ###### `nano /etc/sudoers`  → After entering sudoers with the command, type your username in the field indicated by the red arrow to allow root (administrator) privileges.
    * ###### ALL= → Specifies which users can use Sudo permissions. (All users can use it.)
    * ###### (ALL : ALL) → Indicates which terminal (tty) and which user is authorized. The first "ALL" authorizes command execution from any terminal. The second "ALL" authorizes any target user.
      * ###### "tty" → Usually specifies the name of the device file for the command or shell the user is currently running.
    * ###### ALL → Specifies which commands can be used. "ALL" allows this authorization to run any command. That is, these users can run all commands without being restricted to a specific command such as "/bin/bash".
  * ###### After granting permissions, save and exit.

  * ###### `sudo vim /etc/ssh/sshd_config`  → Enter the /etc/ssh/sshd_config directory with the command, then change the default port 22 to 4242 (optional, you can make a different port number).
  * ###### `#Port 22`  → This line specifies the listening port of the SSH server. By default, SSH uses port 22. However, it is recommended to change this port number for security purposes. Using a port other than the known port numbers provides an additional security measure against automated bot attacks. For example, you can specify a different port number, such as 2222.
  * ###### `#Port 22`  → Find SSH port ⇒ #Port 22 in the comment line and change the port number to 4242.
  * ###### `Port 4242`  → It will be like this
  * ###### `#PermitRootLogin prohibit-password`  → After changing the port number to 4242, let's go down and find this command line.
  * ###### `#PermitRootLogin prohibit-password`  → This line specifies the policy for SSH login for the root user. prohibit-password prohibits password-based login for the root user and only allows SSH key-based authentication.
  * ###### If you want to completely block the root user from logging in via SSH, you can set the "PermitRootLogin" line to "no" only. In this case, the root user can only log into the system through another user account and then switch to root privileges with the "su" command.
  * ###### In short, if "PermitRootLogin" remains as PermitRootLogin, we cannot log in except root, so we change it to "no" because we will log in as a normal user.
  * ###### The final version should be as follows.
    * ###### `Port 4242`
    * ###### `PermitRootLogin no`
